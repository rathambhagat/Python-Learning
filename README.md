## ⚙️ How We Actually Work

Most people learn alone, get bored, quit.  
We solved that with a system.

### 🏛️ The Discord Constitution

**Mission:** Sirf code karne ka re baba.

#### SECTION 1: GENERAL LAWS

| Rule | What It Means |
|------|--------------|
| 🚫 No Code Vomit | Don't paste 10+ lines in chat. Use GitHub links. Penalty: the other person will touch you. |
| 🌅 Daily Ritual | Morning: post goal in `#daily-goals` before 10 AM. Night: post proof in `#end-of-the-day-report` before sleeping. |
| 🇬🇧 English + Hinglish | Talk however, but variables and code comments = professional English. Recruiters don't read Hinglish. |

#### SECTION 2: THE 4 VILLAINS (Crisis Protocols)

```
Villain 1: The Flat Tire (Sickness)
→ Healthy person keeps going, slower pace.
→ Sick person returns and codes fast to catch up.
→ No guilt trips.

Villain 2: The Speed Gap (One person learns faster)
→ Faster person becomes "Senior Dev."
→ Job: Teach, not mock. Teaching cements your own knowledge.
→ Pull your bro up. Don't leave him behind.

Villain 3: The College Attack (Exams)
→ PAUSE heavy learning. DO NOT stop completely.
→ Maintenance Mode: 1 Easy LeetCode problem per day. 15 mins.
→ No Zero Days.

Villain 4: The Ghosting (Burnout)
→ Stop the syllabus immediately.
→ Build something stupid and fun for 2 days.
→ (a spam bot, a meme generator, anything)
→ Remind the brain that coding is a superpower, not homework.
```

### 📊 Daily Tracking System

Every day, both of us post in Discord:

**Morning post format (`#daily-goals`):**
```
DATE: [date]
TODAY'S GOAL: [specific task with link if LeetCode]
ESTIMATED TIME: [X hours]
```

**Night post format (`#end-of-the-day-report`):**
```
DATE: [date]
COMPLETED: ✅ / ❌
WHAT I BUILT/SOLVED: [description + GitHub link]
BLOCKERS: [what was hard]
TOMORROW: [next task]
```

**Status:** ✅ COMPLETE  
**Duration:** ~4 weeks

### What We Covered

| Part | Topics | Code |
|------|--------|------|
| Part 1: Building Blocks | Variables, Data Types, Operators, Input/Output | 
| Part 2: Control Flow | If-Else, For/While Loops, Break/Continue | 
| Part 3: Functions | def, Parameters, Return, Scope, Modules |

### Key Takeaways
- Dictionaries are genuinely critical — every Phase 2 DSA pattern uses them
- OOP clicked once we stopped thinking of classes as "templates" and started thinking of them as blueprints for objects that have memory
- CS50P's problem sets are harder than they look — don't skip them
